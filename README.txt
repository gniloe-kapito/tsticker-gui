                            tsticker-gui — Telegram Sticker Manager
══════════════════════════════════════════════════════════════════════════════

Что это:
  Десктопное приложение для управления стикерпаками Telegram.
  Заливай, редактируй, синхронизируй стикеры — мышкой, без консоли.


──────────────────────────────────────────────────────────────────────────────
  БЫСТРЫЙ СТАРТ (3 шага)
──────────────────────────────────────────────────────────────────────────────

  1. Установи Python 3.12+ с https://www.python.org/downloads/windows/
     ВАЖНО: при установке поставь галочку «Add python.exe to PATH».

  2. Дважды кликни на «Запустить.bat».
     • При первом запуске он сам создаст окружение в deps\ и поставит
       библиотеки (1-3 минуты, нужен интернет).
     • Появится окно приложения.

  3. В окне приложения открой вкладку «Login» и вставь:
     • Bot token — от @BotFather (формат 1234567890:ABC-DEF...)
     • User id — твой личный Telegram id (от @getidsbot)
     Нажми «Save & log in».


──────────────────────────────────────────────────────────────────────────────
  ЕСЛИ НЕ ЗАПУСКАЕТСЯ
──────────────────────────────────────────────────────────────────────────────

  Если «Запустить.bat» закрывается и GUI не появляется:

  1. «Запустить.bat» теперь сам откроет crash.log в Блокноте, если
     GUI упал при старте — больше не «ничего не произошло».

  2. Дважды кликни на «Debug.bat» — он запустит приложение с открытой
     консолью, где будет видна ошибка.

  3. Файл «crash.log» в этой папке — полный текст ошибки (открой в Блокноте).

  4. Файл «app.log» — постоянный лог приложения (ротация 2.5 МБ).


──────────────────────────────────────────────────────────────────────────────
  ФАЙЛЫ В ЭТОЙ ПАПКЕ
──────────────────────────────────────────────────────────────────────────────

  Запустить.bat        — главный ярлык (двойной клик)
  Debug.bat            — запуск с консолью (для отладки, если не работает)
  setup.bat            — ручная установка deps (Запустить.bat сам его зовёт)
  Удалить всё.bat      — удаляет deps\ (окружение ~80 МБ)
  README.txt           — этот файл
  launch.py            — точка входа (запускается bat-файлами)
  crash.log            — создаётся при краше (открой в Блокноте)
  app.log              — постоянный лог приложения (создаётся при запуске)
  app\                 — код приложения (не трогать)
  deps\                — виртуальное окружение (создаётся автоматически)


──────────────────────────────────────────────────────────────────────────────
  КАК ПОЛЬЗОВАТЬСЯ
──────────────────────────────────────────────────────────────────────────────

  1. Создай новый стикерпак:
     • Вкладка «Init» → имя пакета, заголовок, тип (обычно regular) →
       «Initialise». Появится папка с index.json и пустой stickers/.
     • Имя пака проверяется на лету — если оно уже занято в Telegram,
       приложение сразу скажет «используй Trace».

  2. Добавь стикеры:
     • Вкладка «Push/Sync» → выбери рабочую папку.
     • Перетащи картинки мышкой прямо в окно (png/jpg/gif/webp/webm/mov/mp4).
       Или нажми «Add files…».
     • Видишь миниатюры всех стикеров в пакете.

  2б. Прикрепи emoji к стикерам:
     • Дважды кликни по стикеру в сетке → откроется диалог выбора emoji.
     • Или выдели несколько стикеров и нажми «Set emoji…».
     • В диалоге: 9 категорий (Smileys, Animals, Food, …), поиск по слову
       (cat, heart, fire), вкладка Recent с последними использованными.
     • Можно напечатать/вставить emoji вручную.
     • Назначенные emoji показываются прямо на миниатюре (badge) и
       используются при Push ВМЕСТО тех что в имени файла.
     • Emoji сохраняются в файл .sticker_emojis.json рядом с index.json.

  3. Залей в Telegram:
     • Нажми «Push now». Готово — стикеры в Telegram!
     • Ссылка на пак: https://t.me/addstickers/<имя_пакета>_by_<имя_бота>

  3б. Поменять emoji на УЖЕ залитых стикерах:
     • Если стикеры уже в Telegram, но emoji не те — НЕ нужно перезаливать.
     • Дважды кликни по стикеру → назначь новый emoji → «Save emoji».
     • Повтори для каждого стикера которому надо сменить emoji.
     • Нажми «Sync emoji now» (третья карточка внизу).
     • Приложение вызовет Telegram setStickerEmojiList для каждого стикера
       с новым emoji — картинка не перезаливается, меняется только emoji.
       Быстро (~2 сек на стикер из-за rate limit Telegram).

  4. Импорт/экспорт emoji:
     • «Export emoji…» — сохранить emoji-маппинг пака в .json.
     • «Import emoji…» — загрузить маппинг из .json (merge с существующим).
     • Удобно для копирования emoji между пакетами.

  5. Скачать чужой пак:
     • Вкладка «Download» → вставь ссылку t.me/addstickers/... → «Download».
     • Или «Trace» — если пак создан твоим ботом (можно редактировать).

  6. Синхронизировать с облаком:
     • Вкладка «Push/Sync» → «Sync now» — скачает свежую версию из Telegram.

  7. Посмотреть инфо о паке:
     • Вкладка «Show» → выбери папку → «Refresh».
     • Показывает локальный index.json, состояние в облаке, количество
       снапшотов. Кнопка «Restore snapshot» — восстановить stickers/ из
       последнего бэкапа (если что-то пошло не так).

  8. Восстановление после сбоя:
     • Если index.json оказался пустым (краш при push) — приложение
       подскажет сколько есть снапшотов и предложит Restore/Sync.
     • Правый клик по стикеру → контекстное меню (Set emoji / Open / Refresh).


──────────────────────────────────────────────────────────────────────────────
  ТРЕБОВАНИЯ
──────────────────────────────────────────────────────────────────────────────

  • Windows 10/11 (64-bit)
  • Python 3.12+ (с галочкой "Add to PATH" при установке)

  ОПЦИОНАЛЬНО (для видео/анимации):
  • ffmpeg — https://ffmpeg.org/download.html
    Нужен только для конвертации GIF/WebM/MP4 в анимированные стикеры.
    Статичные PNG/JPG стикеры работают БЕЗ ffmpeg.

  ImageMagick больше НЕ нужен! В этой версии используется Pillow
  (чистый Python, без системных зависимостей) для обработки картинок.


──────────────────────────────────────────────────────────────────────────────
  ОСОБЕННОСТИ ВЕРСИИ
──────────────────────────────────────────────────────────────────────────────

  • Прикрепление emoji к стикерам: дважды кликни по стикеру в сетке
    или выдели несколько и нажми «Set emoji…». 9 категорий + поиск +
    Recent. Назначенные emoji перекрывают те что в имени файла.

  • Смена emoji на уже залитых стикерах: кнопка «Sync emoji now» —
    вызывает Telegram setStickerEmojiList, картинка не перезаливается.

  • Импорт/экспорт emoji-маппинга: кнопки «Import emoji…» / «Export emoji…».

  • Батч-заливка: можно заливать больше 30 стикеров за раз (до 120).
    Приложение само разобьёт загрузку на порции по 25 штук.

  • Атомарная запись index.json: файл никогда не останется пустым
    даже при краше приложения mid-write.

  • UTF-8 везде: исправлены все места чтения index.json — больше никаких
    "'charmap' codec can't decode byte 0x98" на emoji или кириллице.

  • Подсказки (tooltips): наведи курсор на любое поле — увидишь
    подробное объяснение что туда вводить.

  • Живая валидация: в Init tab поле pack_name проверяется на лету
    (формат + доступность имени на Telegram, если залогинен).

  • Восстановление из снапшотов: во вкладке Show есть кнопка
    «Restore snapshot» — возвращает stickers/ к состоянию до сбоя.

  • Status bar: в правом нижнем углу показывается что происходит
    (Idle / Uploading 3/25 / Downloading 5/76 / Logged in @bot).

  • Drag-and-drop: перетащи картинки прямо в окно Push/Sync.
    Они автоматически скопируются в папку stickers/.

  • Контекстное меню: правый клик по стикеру → Set emoji / Open folder /
    Refresh.

  • Лог в файл: все сообщения дублируются в app.log рядом с launch.py
    (ротация 2.5 МБ, хранятся последние 2 файла).

  • Запоминает последнюю рабочую папку между запусками.

  • Надёжный запуск: «Запустить.bat» сам открывает crash.log в
    Блокноте если GUI упал при старте.
